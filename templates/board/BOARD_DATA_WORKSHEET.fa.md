# کاربرگ اطلاعات واقعی مجتمع

این کاربرگ مانع طراحی و ساخت نیست. تا زمان دریافت پاسخ، سامانه از `config/lavizan.defaults.json` با برچسب «پیشنهادی/تأییدنشده» استفاده می‌کند. پاسخ مصوب در یک فایل override خصوصی وارد و پیش از پایلوت اعتبارسنجی می‌شود.

## شیوه پاسخ

برای هر ردیف یکی از گزینه‌های «تأیید»، «اصلاح» یا «فعلاً نامشخص» را مشخص کنید. اطلاعات هویتی ساکنان، شماره تماس، اسناد مالی و رمزها را در این فایل یا GitHub ننویسید.

## اقلامی که جدا و امن تحویل می‌شوند

- فهرست واحدها با شناسه خارجی، بلوک، طبقه، شماره و وضعیت؛
- اشخاص و رابطه سکونت در فایل جداگانه و رمزگذاری‌شده؛
- مانده افتتاحیه و دوره‌های مالی با تأیید حسابدار؛
- کارکنان، شیفت‌ها، امکانات، قراردادها و قوانین مصوب؛
- لوگو و متن‌های رسمی اعلان؛
- امضاها و صورت‌جلسه تأیید داده.

دریافت → بررسی ویروس و checksum → محیط staging → گزارش خطا → تأیید متولی داده → import تکرارپذیر → تطبیق → امضا.
