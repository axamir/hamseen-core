# Project Rules

1. MVP scope is fixed by document 16; additions require a recorded decision.
2. English and Persian product documents change together.
3. No secrets, real resident records, or copied production exports enter Git.
4. All tenant data access is scoped and authorization-tested.
5. Financial and governance records are corrected through traceable transitions, not silent mutation.
6. External services and model providers are adapters behind Hamseen APIs.
7. UI must meet responsive, RTL, keyboard, contrast, and reduced-motion requirements.
8. A feature is not complete without failure states, tests, audit behavior, and documentation.
