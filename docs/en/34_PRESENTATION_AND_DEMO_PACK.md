# 34 — Presentation, Landing, and Product Demo Pack

## Audience and outcome

| Audience | Must understand | Final action |
|---|---|---|
| Board | control, transparency, risk, pilot path | validation meeting |
| Resident | easier daily work and protected privacy | join testing |
| Guard/manager | clear, traceable work queue | evaluate flow |
| Developer/operator | housing differentiation and scalable operations | request demo/pilot |
| Investor | problem, model, moat, evidence, plan | diligence conversation, not return promise |

## Landing narrative

1. Hero: “Hamseen — Good neighbors are a blessing” and Community OS definition.
2. Before: calls, paper, scattered groups, unclear follow-up.
3. Familiar hub: each capability a mini app, one account, role-aware access.
4. A real day: guest → package → charge → ticket → booking.
5. Interactive role switch: resident, guard, manager, accountant.
6. Lavizan pilot; unconfirmed data labeled Demo.
7. Community/local economy as next phase, not built claim.
8. AI with source, permission, and human approval.
9. Security, privacy, no data sale.
10. Roadmap with honest Built/Building/Future state.
11. Separate CTAs for pilot, feedback, partner, investor.

## Interactive demo screens

`LAND-01`, `HUB-01`, `GUEST-01`, `GUARD-01`, `FIN-01`, `TKT-01/02`, `RSV-01`, `AI-01`.

All data is deterministic synthetic data. User switches role/language, performs sample action, sees state/notification, and can return to the story without dead ends.

## Twelve-slide pitch

Cover; Problem; Why now; Solution; Product tour; Users/buyer; Market evidence; Business model; Competition/moat; real Pilot/traction; Roadmap/team; Ask/use of funds. Market claims and figures require dated sources; fabricated traction is prohibited.

## Deliverables

- bilingual mobile-first local web experience
- deep links and meta/social preview
- two-page summary PDF and pitch deck
- mobile/tablet/desktop screenshots
- five- and fifteen-minute talk tracks
- consented minimum-data interest form

## Quality

True RTL/LTR, WCAG AA, 44px touch targets, no horizontal overflow, LCP target under 2.5 seconds on suitable network, reduced motion, keyboard navigation, no fabricated claims, explicit product status.

## Measurement

Privacy-safe events: chapter view, role/language switch, tour start/end, feature open, summary download, interest submit. No fingerprinting or analytics before consent.
