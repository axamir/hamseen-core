# 12 — AI Intelligence Layer

AI is a controlled platform capability, not an ornamental chatbot.

```text
App / Telegram / Bale
        ↓
Hamseen Core API
        ↓
Permission + Privacy Context
        ↓
AI Gateway
        ↓
OpenAI / Gemini / Local / Custom Provider
        ↓
Read, Draft, Recommend, or Approval-Gated Action
```

## Role assistants

- Resident: charges, reservations, guests, tickets, services, notice summaries
- Guard: voice-assisted guest/package records, permits, shift summary
- Manager: smart inbox, prioritization, assignment drafts, daily brief
- Accountant: variance analysis, debtor list, duplicate warnings, report drafts
- Board: governance summaries, trend analysis, meeting minutes drafts
- Provider: order summaries, overdue work, demand and inventory insights

## AI Control Center

Provider/model selection, encrypted credentials, task routing, cost/usage limits, prompt versions, allowed data scopes, error monitoring, feature switches, audit history, and emergency disable.

## Safety

- AI access equals user access.
- Retrieval uses authoritative Hamseen data; uncertain answers are labeled.
- Payments, permission changes, deletions, formal notices, and governance decisions require explicit human confirmation.
- Every request records actor, model, data scope, output, approval, and execution.
