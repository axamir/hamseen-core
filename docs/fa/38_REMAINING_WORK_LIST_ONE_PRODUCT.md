# ۳۸ — فهرست فنی اول: تکمیل محصول

تاریخ وضعیت: ۲۶ تیر ۱۴۰۵. «انجام‌شده» یعنی در برش محلی محدود وجود دارد و آزموده شده، نه اینکه production استقرار یافته باشد.

| شناسه | خروجی | وضعیت | شاهد / شرط پایان |
|---|---|---|---|
| P1 | هاب مشترک مبتنی بر نقش | انجام‌شده | رابط و آزمون ساکن/نگهبان/مدیر/حسابدار |
| P2 | چرخه مهمان | انجام‌شده | دعوت، تأیید، ورود، خروج، تعارض نسخه |
| P3 | امانت بسته | انجام‌شده | دریافت، قصد اعلان، تحویل، حسابرسی |
| P4 | درخواست | برای برش MVP انجام‌شده | ایجاد ساکن، ارجاع/شروع مدیر، آزمون نقش |
| P5 | هسته رزرو | برای برش MVP انجام‌شده | رزرو قطعی و رد تداخل |
| P6 | مالی پایه ساکن | برای برش MVP انجام‌شده | مانده مصنوعی، فیش، کنترل تکرار، بررسی |
| P7 | انتشار اعلان | انجام‌شده | پیش‌نویس، انتشار، نسخه، قصد اعلان |
| P8 | AI محدود | انجام‌شده | منبع و پاسخ پیشنهادی مبتنی بر نقش، audit |
| P9 | ارائه واکنش‌گرا دو‌زبانه | انجام‌شده | لندینگ/تور و آزمون ساختاری فارسی/انگلیسی |
| P10 | فرض‌های قابل جایگزینی | انجام‌شده | پیش‌فرض معتبر، override، کاربرگ هیئت‌مدیره |
| P11 | شروع ورود داده | انجام‌شده | نمونه CSV غیرشخصی و مسیر امن مستند |
| P12 | همه وضعیت‌های UI، رسانه و آفلاین | تعریف شده؛ پیاده‌سازی مانده | پیش از پایلوت باید سند ۱۷ پاس شود |
| P13 | حسابداری جامع دوطرفه | برای MVP production مانده | دفتر متوازن، برگشت و تطبیق آزموده |
| P14 | احراز هویت و عضویت واقعی | مانده | OTP/MFA/session و آزمون مجوز |

P12 تا P14 اولین backlog برای Cursor/TREA است. مارکت کامل، معامله ملک، اپ native، IoT و AI پیش‌بین خارج از MVP‌اند و نباید هسته پایلوت را عقب بیندازند.
