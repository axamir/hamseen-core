# ۳۵ — وضعیت پیاده‌سازی و وابستگی‌ها

تاریخ بازبینی: ۲۶ تیر ۱۴۰۵ / ۱۷ ژوئیه ۲۰۲۶

## ساخته و آزموده‌شده

- اسناد دو‌زبانه ۰۰ تا ۳۵ و قراردادهای اجرایی MVP
- لندینگ و تور تعاملی دو‌زبانه با چهار نقش و داده مصنوعی
- MVP محلی با SQLite و API نقش‌محور
- هاب ساکن، نگهبان، مدیر و حسابدار
- دعوت/تأیید/ورود/خروج مهمان
- ثبت و ارجاع درخواست
- رزرو با جلوگیری از تعارض
- مانده مصنوعی، ثبت فیش، جلوگیری از تکرار و بررسی حسابدار
- Audit برای اقدامات حساس و آزمون cross-tenant
- دریافت/اعلان/تحویل بسته و Notification Intent
- ساخت و انتشار نسخه‌دار اعلان مدیریت
- دستیار محدود نقش‌محور با منبع و بدون اجرای خودکار
- پیش‌فرض معتبر و صریحاً فرضی لویزان با مسیر override خصوصی
- کاربرگ ساده دو‌زبانه هیئت‌مدیره و نمونه امن ورود داده
- CI کیفیت، پایه Compose وابستگی محلی و قرارداد متغیر محیط
- هدر امنیت، شناسه درخواست، کنترل حجم/نرخ، سلامت/آمادگی و توقف نرم
- backup/checksum/integrity برای SQLite و starter مربوط به RLS/audit در PostgreSQL

## هنوز Production-ready نیست

- احراز هویت واقعی، OTP/MFA و مدیریت نشست
- پیاده‌سازی کامل API با PostgreSQL/NestJS، صف، مشاهده‌پذیری و ذخیره فایل زنده
- تحویل واقعی Push/Bale/Telegram/SMS (Intent داخلی ساخته شده است)
- فایل و تصویر، جزئیات پیشرفته بسته و گزارش مالی جامع
- backup/restore رمزگذاری‌شده production، استقرار و آزمون بار/دسترسی/نفوذ
- متن حقوقی و قراردادهای نهایی

## وابستگی خارجی — هیئت‌مدیره

پرسش‌نامه ۲۶، داده واحد/ساکن، قواعد مالی، مهمان، امکانات، retention، مسئولان پایلوت، بودجه و امضای پذیرش. نبود پاسخ مانع توسعه با داده مصنوعی نیست، اما مانع استقرار واقعی است.

سند ۳۷ و کاربرگ هیئت‌مدیره، پیش‌فرض امن و مسیر override می‌دهند؛ بنابراین نبود پاسخ دیگر ساخت را متوقف نمی‌کند، اما تا تأیید نهایی مانع پایلوت زنده است.

## وابستگی خارجی — متخصصان

- حقوقی/حریم خصوصی: مبنای پردازش، retention، قرارداد داده، شرایط خدمت
- حسابداری/مالیاتی: فرمول و دفتر، ریال/تومان، صورتحساب، درگاه
- امنیت: بازبینی تهدید، آزمون نفوذ و Incident plan
- برند/بازار: نام/دامنه/هویت نهایی و اعداد معتبر بازار

## معیار تبدیل به پایلوت

انتقال معماری تولید، تکمیل جریان‌های MVP سند ۱۶، Gate امنیت سند ۳۱، rehearsal داده و امضای سند ۳۲. وضعیت «ساخته‌شده» در ارائه فقط برای موارد بخش اول استفاده می‌شود.
