# ۳۳ — مدل تجاری، قیمت‌گذاری و اقتصاد محصول

## اصل مدل

مشتری پرداخت‌کننده اصلی، مجتمع/شرکت مدیریت/سازنده است؛ ساکن برای هسته ضروری مجبور به خرید اشتراک جداگانه نمی‌شود. قیمت نهایی فقط پس از پایلوت و اندازه‌گیری هزینه ارائه تصویب می‌شود.

## بسته‌های پیشنهادی

| بسته | مناسب | شامل | مبنای قیمت |
|---|---|---|---|
| Pilot | یک بلوک یا ۲۰–۴۰ واحد | راه‌اندازی محدود، آموزش، هسته عملیات، گزارش پایلوت | مبلغ ثابت زمان‌دار |
| Core | مجتمع کوچک/متوسط | واحد، اعضا، اعلان، مهمان، بسته، درخواست، رزرو پایه | حداقل ماهانه + واحد فعال |
| Operations | مجتمع بزرگ | Core + مدیریت، قرارداد، گزارش، حسابداری پایه، کانال | واحد فعال + ماژول |
| Platform | سازنده/مدیر چندمجتمعی | چند tenant، برند، API، SLA، گزارش تجمیعی | قرارداد سالانه |

## درآمدهای مجاز

- هزینه راه‌اندازی، پاک‌سازی داده و آموزش
- اشتراک SaaS مجتمع بر اساس واحد فعال و ماژول
- پشتیبانی/SLA و سفارشی‌سازی کنترل‌شده
- کارمزد شفاف تراکنش خدمات یا ماژول‌های شریک، فقط با قرارداد
- اتصال سازمانی، گزارش سفارشی و White-label

فروش داده شخصی، تبلیغ پنهان، وعده درآمد ساکن یا ROI تضمینی مدل همسین نیست.

## فرمول سناریوی قیمت

```text
MRR = base_platform_fee
    + active_units × per_unit_fee
    + enabled_paid_modules
    + support_tier
```

سه سناریو `Conservative / Base / Growth` در مدل مالی استفاده می‌شود. اعداد قبل از دریافت قیمت زیرساخت، حقوق تیم، مالیات، پشتیبانی و نرخ تبدیل فقط placeholder هستند.

## اقتصاد واحد

| شاخص | فرمول |
|---|---|
| ARPA | درآمد ماهانه / مجتمع پرداخت‌کننده |
| Gross margin | (درآمد - زیرساخت - پیام - پشتیبانی مستقیم) / درآمد |
| CAC | هزینه فروش و استقرار مشتریان جدید / مشتری جدید |
| Payback | CAC / سود ناخالص ماهانه مشتری |
| Logo churn | مجتمع از دست‌رفته / مجتمع ابتدای دوره |
| NRR | درآمد تکرارشونده پایان با توسعه/کاهش / درآمد ابتدای مشتریان موجود |
| Support cost/unit | هزینه مستقیم پشتیبانی / واحد فعال |

هدف عددی فقط پس از داده پایلوت تعیین می‌شود.

## هزینه‌ها

- توسعه محصول، طراحی، امنیت و QA
- hosting، دیتابیس، storage، backup، monitoring
- پیامک/Push/Bale/Telegram و AI provider
- onboarding، مهاجرت، آموزش و پشتیبانی
- فروش، قرارداد، حسابداری، حقوقی و بیمه
- ذخیره رخداد، بازیابی و SLA

## مسیر فروش

Lead → جلسه کشف → دموی نقش‌محور → ارزیابی داده/فرایند → پیشنهاد پایلوت → قرارداد و DPA → پایلوت → گزارش ارزش → توسعه قرارداد. معیار فروش «کاهش اصطکاک قابل اندازه‌گیری و اعتماد» است، نه فهرست بلند قابلیت‌ها.

## سرمایه‌گذاری

Use of funds پیشنهادی: محصول/مهندسی، امنیت و زیرساخت، پایلوت و پشتیبانی، فروش سازمانی، ذخیره قانونی/عملیاتی. هیچ سندی بازده تضمینی یا ارزش‌گذاری ساختگی ارائه نمی‌کند.

## تصمیم‌های خارجی باز

شخصیت حقوقی، مالیات، روش صورتحساب، درگاه، نرخ پیام، قرارداد داده، مالکیت IP، شرایط خدمات و قیمت نهایی نیازمند تأیید حقوقی/مالی و داده بازار هستند.
