# ۰۳ — نقش‌ها و دسترسی‌ها

همه افراد یک هاب پایه مشترک دارند. نقش‌ها مینی‌اپ‌ها و مجوزهای کاری را اضافه می‌کنند و محصول جداگانه نمی‌سازند.

| نقش | قابلیت معمول | محدودیت روشن |
|---|---|---|
| ساکن | واحد خود، اعلان، مهمان، درخواست، رزرو، خدمات و مارکت | بدون دسترسی به داده خصوصی یا مالی واحد دیگر |
| مالک | قابلیت ساکن و سوابق واحد تحت مالکیت | بدون پیام‌های خصوصی مستأجر |
| مستأجر | دسترسی ساکن در محدوده سکونت | بدون اسناد مختص مالکیت |
| نگهبان | مهمان، رخداد ورود، بسته و گزارش شیفت | بدون بدهی واحدها یا مالی هیئت‌مدیره |
| مدیر داخلی | عملیات، ارجاع، اعلان و خدمات‌دهنده | بدون اقدام مالی برگشت‌ناپذیر خارج از اختیار |
| حسابدار | شارژ، پرداخت، هزینه و گزارش | بدون گفت‌وگوی خصوصی یا کنترل ورود |
| هیئت‌مدیره | حاکمیت، گزارش، تأیید و رأی | مطابق سیاست مصوب هیئت‌مدیره |
| خدمات‌دهنده | کاتالوگ، سفارش، برنامه و تسویه خودش | بدون داده نامرتبط ساکن یا مجتمع |
| سوپرادمین | راه‌اندازی مجتمع، ماژول، نقش، اتصال و حسابرسی | دسترسی اضطراری استثنایی و ثبت‌شونده |

## قواعد

- پیش‌فرض عدم دسترسی است و حداقل مجوز لازم داده می‌شود.
- یک کاربر می‌تواند چند نقش داشته باشد.
- هر تغییر حساس یک رخداد حسابرسی تغییرناپذیر می‌سازد.
- دسترسی AI دقیقاً برابر دسترسی کاربر آغازکننده است، نه بیشتر.
- حذف مالی، تغییر مجوز و خروجی حریم خصوصی تأیید تقویت‌شده می‌خواهد.
