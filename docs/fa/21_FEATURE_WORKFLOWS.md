# ۲۱ — گردش‌کار قابلیت‌های حیاتی

هر گردش‌کار حالت روشن، کنترل مجوز، اعلان و رخداد حسابرسی دارد.

| جریان | حالت‌ها | استثناهای الزامی |
|---|---|---|
| عضویت | دعوت → تأیید هویت → درخواست واحد → تأیید/رد → فعال/تعلیق | فرد تکراری، دعوت منقضی، اختلاف مالک/مستأجر |
| مهمان | پیش‌نویس → دعوت/انتظار → تأیید/رد/انقضا → ورود → خروج/بسته | بی‌پاسخی، گیت آفلاین، پیمانکار، پیک، مهمان دوره‌ای |
| بسته | دریافت → اعلان → تأیید دریافت پیام → تحویل/مرجوع | واحد اشتباه، آسیب، تأخیر، ثبت آفلاین |
| درخواست | پیش‌نویس → ثبت → دسته‌بندی → ارجاع → انجام → انتظار → حل → تأیید/بسته/بازگشایی | فوریت، تکرار، نقض SLA، رد راه‌حل |
| فیش | پیش‌نویس → ثبت → بررسی → تأیید/رد → سند قطعی | تکرار، مبلغ/دوره اشتباه، تصویر ناخوانا، برگشت |
| رزرو | آزاد → نگه‌داری → قطعی → ورود → پایان/لغو/عدم حضور | تعارض ظرفیت، تعمیرات، استرداد، نیاز به تأیید |
| سفارش | سبد → ثبت → قبول/رد → آماده‌سازی → آماده/ارسال → تحویل → بسته/اختلاف | جایگزینی، تغییر قیمت، لغو، تحویل ناقص |
| اعلان | پیش‌نویس → بازبینی → زمان‌بندی/انتشار → مشاهده → بایگانی | اضطراری، مخاطب هدف، اصلاح/نسخه |
| رأی | پیش‌نویس → قفل واجدان → باز → بسته → شمارش → تأیید/انتشار | نبود حد نصاب، ممتنع، بازشماری، ابطال حقوقی |

## الزامات مشترک

- هر تغییر، کاربر، زمان، حالت قبل/بعد، کانال و دلیل را ثبت می‌کند.
- تکرار فرمان نباید پرداخت، سفارش، دعوت یا اعلان تکراری بسازد.
- کاربر وضعیت ساده و اقدام بعدی را می‌بیند.
- اقدام مخرب یا رسمی تأیید و اختیار مناسب می‌خواهد.
