# ۳۲ — بسته عملیات، مهاجرت و سنجش پایلوت

## مالکیت اجرایی

| نقش | مسئولیت |
|---|---|
| حامی هیئت‌مدیره | اختیار، بودجه، رفع مانع و پذیرش نهایی |
| مالک محصول مجتمع | تصمیم روزانه، هماهنگی و اولویت بازخورد |
| مدیر پایلوت همسین | برنامه، انتشار، ریسک و گزارش |
| Data Steward | دریافت، تطبیق و تأیید داده مهاجرت |
| مسئول امنیت/حریم | دسترسی، رخداد و تأیید Gate |
| پشتیبانی | آموزش، تیکت و ارتباط کاربران |

## مراحل و Gate

1. **Discovery:** پرسش‌نامه ۲۶، نقشه فرایند فعلی، انتخاب کاربران و امکان رزرو.
2. **Demo:** پروتوتایپ با داده مصنوعی؛ تأیید جریان توسط مدیر، نگهبان، حسابدار و ساکن.
3. **Data rehearsal:** import آزمایشی، گزارش خطا، تطبیق نمونه و پاک‌سازی محیط.
4. **Controlled launch:** ۲۰–۴۰ واحد، یک شیفت، یک مدیر، یک دوره مالی بسته، یک امکان.
5. **One block:** پس از دو چرخه پایدار و بدون مغایرت بحرانی.
6. **Full site:** موجی، با rollback و support capacity.

هیچ Gate با تصمیم شفاهی رد نمی‌شود؛ چک‌لیست، مالک و امضا دارد.

## قالب مهاجرت

فایل‌های جدا: `units`, `people`, `occupancies`, `vehicles`, `opening_balances`, `charges`, `receipts`, `amenities`, `staff`, `contracts`. هر فایل دارای شناسه خارجی، منبع، وضعیت تأیید و تاریخ snapshot است. PII از مسیر امن منتقل می‌شود، نه ایمیل عمومی یا Git.

جریان: دریافت → antivirus/checksum → staging → schema validation → duplicate match → گزارش اختلاف → تأیید Data Steward → import idempotent → reconciliation → sign-off.

## آموزش و پشتیبانی

- ساکن: راهنمای ۵ دقیقه‌ای و کارت «مهمان، فیش، درخواست، رزرو».
- نگهبان: تمرین سناریویی ورود، بسته، آفلاین و تحویل شیفت.
- مدیر/حسابدار: محیط staging، اصلاح امن، گزارش و escalation.
- پشتیبانی: ساعت پاسخ، کانال واحد، FAQ نسخه‌دار و incident escalation.

## KPI و فرمول

| شاخص | تعریف |
|---|---|
| Activation | خانوار دارای ورود و یک اقدام / خانوار دعوت‌شده |
| Weekly active household | واحد با حداقل یک رویداد معنادار در ۷ روز |
| Guest decision time | میانه زمان درخواست تا پاسخ |
| Package custody time | میانه دریافت تا تحویل |
| Ticket resolution | میانه ثبت تا حل، جدا بر اساس فوریت |
| Receipt review | میانه ثبت فیش تا تصمیم |
| Reconciliation accuracy | ۱ - قدرمطلق مغایرت / مبلغ مورد انتظار |
| Support burden | تیکت پشتیبانی به ازای ۱۰۰ خانوار فعال |
| Task success | تکمیل بدون کمک در تست کاربردپذیری |
| Critical incident | رخداد SEV-1/2؛ هدف پایلوت صفر |

هدف عددی فقط پس از baseline هفته صفر تصویب می‌شود.

## عملیات روزانه و هفتگی

- روزانه: سلامت سرویس، صف شکست‌خورده، backup، رخداد، مغایرت و کارهای معوق.
- هفتگی: KPI، بازخورد نقش‌ها، خطاهای پرتکرار، ریسک، تصمیم تغییر و release plan.
- هر تغییر دامنه/قاعده با Decision ID ثبت می‌شود؛ hotfix پس از رخداد مستندسازی می‌شود.

## Rollback و توقف

توقف در رخداد امنیتی جدی، مغایرت مالی حل‌نشده، از دست رفتن داده، نبود مالک عملیاتی یا اختلال ایمنی گیت. روش قبلی تا قبولی دو چرخه حیاتی موازی می‌ماند. rollback شامل توقف فرمان، حفظ شواهد، export تغییرات و بازگشت کنترل‌شده است.

## پذیرش

گزارش تطبیق داده، نتیجه تست امنیت/بازیابی، KPI، مشکلات باز، آموزش انجام‌شده، مالک پشتیبانی و امضای هیئت‌مدیره/مدیر/حسابدار ضمیمه می‌شود.
